# 轮臂机器人安卓测试软件开发方案（Windows）

## 1. 目标
在 Windows 环境下开发一款 Android 测试 App，实现：
- 手动输入或选择机器人 IP 并建立连接。
- 下发测试指令（如底盘移动、机械臂关节动作、急停、复位）。
- 实时查看机器人状态（在线状态、关节角度、电量、故障码）。
- 保存测试日志，便于回溯问题。

## 2. 推荐技术栈
- 开发环境：Windows 10/11 + Android Studio（最新版）
- 语言：Kotlin
- 架构：MVVM + Repository
- 网络通信：
  - 首选 TCP Socket（若机器人控制器提供 TCP 指令接口）
  - 备选 HTTP/REST（若控制器提供 Web API）
- 并发：Kotlin Coroutines + Flow
- 本地存储：Room（记录测试日志）
- UI：Jetpack Compose（效率高，便于快速迭代测试页面）

## 3. App 核心模块

### 3.1 连接管理模块
- 输入 IP 和端口，支持“最近连接”列表。
- 提供连接状态机：未连接 / 连接中 / 已连接 / 断开 / 重连中。
- 心跳机制（如每 1~2 秒发送心跳包）用于判断在线状态。

### 3.2 指令下发模块
建议采用统一报文格式（JSON 示例）：

```json
{
  "cmd": "arm_move",
  "request_id": "20260227123000",
  "payload": {
    "joint": 3,
    "angle": 45.0,
    "speed": 20
  }
}
```

常用测试指令建议：
- `base_move`：底盘线速度/角速度控制
- `arm_move`：单关节或多关节运动
- `arm_home`：机械臂回零
- `stop`：普通停止
- `estop`：急停（最高优先级）
- `get_status`：拉取状态

### 3.3 状态监控模块
建议状态报文包含：
- 连接状态、控制模式
- 各关节角度/温度/电流
- 底盘速度、电池电量
- 告警与故障码

UI 展示建议：
- 绿色：正常
- 黄色：警告
- 红色：故障/急停

### 3.4 测试用例与日志模块
- 一键执行测试序列（例如：连接→回零→关节扫描→底盘前后移动→停止）。
- 每条指令记录发送时间、ACK、执行结果、耗时。
- 支持导出日志（CSV/JSON）用于问题复盘。

## 4. 通信与安全建议
- 控制指令加序号 `request_id`，便于响应匹配。
- 设置发送超时、重试次数（如超时 500ms，重试 2 次）。
- 急停指令独立通道或最高优先级队列。
- 增加“仿真模式/只读模式”，避免误动作。
- 网络异常时自动发送停止指令（可配置）。

## 5. MVP（最小可用版本）开发排期

### 第 1 周

### 第 2 周

- 完成状态实时显示

### 第 3 周

### 第 4 周

- 修复稳定性问题，打包发布测试版

## 6. Kotlin 通信代码骨架（TCP）

```kotlin
class RobotTcpClient(
    private val ip: String,
    private val port: Int
) {
    private var socket: Socket? = null
    private var writer: BufferedWriter? = null
    private var reader: BufferedReader? = null

    suspend fun connect() = withContext(Dispatchers.IO) {
        socket = Socket(ip, port)
        writer = BufferedWriter(OutputStreamWriter(socket!!.getOutputStream()))
        reader = BufferedReader(InputStreamReader(socket!!.getInputStream()))
    }

    suspend fun send(json: String) = withContext(Dispatchers.IO) {
        writer?.apply {
            write(json)
            write("\n")
            flush()
        }
    }

    suspend fun readLine(): String? = withContext(Dispatchers.IO) {
        reader?.readLine()
    }

    fun close() {
        reader?.close()
        writer?.close()
        socket?.close()
    }
}
```
